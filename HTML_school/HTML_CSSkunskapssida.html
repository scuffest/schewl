Kunskap om CSS

1. Vad är CSS? Hur används det?

Cascading style sheet, användes för att stilisera HTML-kod, styra utseendet och layouten av webbsidor. CSS använder selektorer för att rikta stilmallen till specifika element, Kaskadmodellen avgör vilken stil som ska användas om flera stilar tillämpas på samma element, boxmodellen.

2. Vad är skillnaden mellan inline-stil, inbäddad stil och extern stil i CSS?

Inbäddad CSS är i HTML och då riktar vi det mot element och beskriver vad vi vill göra med den. Inline är att man skriver direkt i taggen. Extern innebär att css koden finns inte html koden men i en separat kod och sedan länkar ihop den med HTML link rel=”stylesheet” type=”text/css” href=”style.css”>

3. Vad är en selektor i CSS och hur används den?

Används för att välja och tillämpa stilar på specifika HTML-element och det finns primärt fem sorters selektorer.

4. Vad är skillnaden mellan klass-selektor och ID-selektor i CSS?

ID är ett specifikt namn, det är speciellt på något sätt, ID blir bara existerat en gång jämfört med klass som kan användas flera gånger. Klass kan man ge till flera element.

5. Vad är kaskadordningen i CSS och hur påverkar den hur stilar tillämpas på element?

ID är ett specifikt namn, det är speciellt på något sätt, ID blir bara existerat en gång jämfört med klass som kan användas flera gånger. Klass kan man ge till flera element.

6. Vad är kaskadordningen i CSS och hur påverkar den hur stilar tillämpas på element?

Följer en viss ordning för att avgöra vilka stilregler som tillämpas på ett element, samt när regler har företräde vid en konflikt.

7. Kaskadmodellen är en algoritm som väljer värde(n) som ska appliceras element enligt

  • Arv (Inheritance) - Om element saknar regel så söks en “förälders” stilregler och appliceras.
  • Relevans (Relevance) Behåller enbart regler som matchar element
  • Ursprung och betydelse (Origin and importance) - Sorterar regler baserat på hur viktiga de är och vem som applicerat dem.
  • Noggrannhet (specificity) - Väljer regler med högre specificitet om relevanta regler har samma ursprung och betydelse.
  • Ordning (order of appearance) - Om specifika regler har samma ursprung och betydelse väljer kaskadmodellen det senast angivna värdet i källkoden.

8. Vad är responsiv webbdesign och hur kan CSS användas för att skapa responsiva layouter?

Man kan använda flexbox och grid layouts. Det är att göra så att hemsidan ser bra ut och så att alla ska kunna navigera hemsidan.

9. Vad är boxmodellen i CSS och vilka element påverkas av den?

Alla HTML-elementet ses som boxar (tänk lådor), innehåll innerst, omges av padding som i sin tur omges av border och sist margin. Använd “float” och “clear” för att positionera.

Längst in är det Content, efter det är padding och padding frigör utrymme runt innehållet och till sist Margin som frigör utrymme till andra boxar utanför innehållet.

10. Vad är "grid layout" och hur skiljer det sig från "flexbox"?

Flexbox utvecklades för responsiv design, det har inbyggt att man kan krympa/växa element beroende på skärmstorleken. Flexbox är en “light version” av grid layouts. Gridlayout kan innehålla både rader och kolumner, till skillnad från flexbox kan gå i en riktning.

11. Vad är skillnaden mellan positionering med "static", "relative", "absolute" och "fixed" i CSS?

  • Relativ: positionerar relativt till sitt ursprungliga läge, men påverkas av andra regler.
  • Fixed: Fast position i förhållande till webbläsarfönstret och påverkas inte av scrolling lämnar inte utrymme på sidan där det “normal” skulle placeras.
  • Absolute: Positionerar relativt till närmaste föräldrars element istället för till webbläsarfönstret. Om ingen förälder finns så positioneras elementet relativt body och följer med scrolling.
